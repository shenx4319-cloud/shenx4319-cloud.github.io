测试任务

针对智慧课堂的“手动评价”测试任务，问题截图太长，暂不展示：
问题描述：教师在进行“手动评价”时，无法进行手动改分，评价栏不呈现AI评价的语句
起初我以为是“课堂AI评价”出现的问题，因为本人之前测试时也会出现类似的问题，即不显示“AI评价”，按照当时的错误思维，确实发现了错误，后来确认后才知道这是另一个课堂功能————“去评价”，之前测试从来没有接触过该功能，好在复现难度较低，与“课堂AI评价”功能有逻辑关系，但是仍然有许多病毒bug存在（即无固定逻辑而产生的bug）

举例：

1.如今天有老师反应，课程中心中“某节课堂”的ppt无法播放，但是自己重新复制上传一下就正常了————其实我平时也会经常遇见类似的bug，有时重新上传就正常，有时自动就修复了
2.如今天有老师反应，输入zcat.cn，进入的却是学生登录的界面，但是输入c.zcat.cn就没问题，————这是很早在测试时就困扰我的问题，因为这个bug出现的情况是随机的（刷新浏览器后输入网址/开无痕输入网址/关闭浏览器后重新输入网址）

除此以外，我有时觉得智慧课堂的功能复杂性跟不上功能的普及性

举例：

1.复杂性：预览报告的生成、课程任务显示分数时的必要操作。
2.普及性：智慧课堂的多功能性，在超喵AI群的（使用教程）文档中并未体现
预览报告中“更新”时间与AI评价模板生成时间的不稳定性（时间巨长，且不知道什么时候能更新好）、前端老师跟我都觉得这里有些麻烦。今天我在测试附小教师的课程时，我发现他根本没有用到这个功能，是不知道如何去用呢？还是觉得太麻烦而不想这么去用呢？

毕设学习

毕设学习：毕设题目是《基于强化学习的城镇开发边界模拟优化与空间规划方案自动生成》，大体意思是，把一个区域的特点（人口密度、地形特点、周边商业、战略规划的目标等等）作为算法的基本数据、再为他提供“约束规则”，例如，基本农田红线不能碰、坡度大于 25 度的地区不宜建设、生态敏感区需要避让。这些是算法的 Environment，再对其提出一个规划问题，此时agent进行训练，很简单的动作——奖励——策略机制，最终他能学会在保护耕地与发展经济之间找到平衡点，从而将土地资源的分配达到最优。

前几天心血来潮，想调用一下API，但是也不知道有没有用，不管了，反正花了十块钱调用了deepseek的api，Gemini老师说够我用一辈子了，调用在之前得到的信访项目上，但是呢他只能回答固定的问题（像个大傻子），也不知道跟我的毕设有没有关系。今天在cursor老师的指导下，成功把毕设的数据全都处理好了（暂时没有遇到什么有含金量的问题），明天争取能让agent训练上。奥对了，今天cursor用冒了，花了40r买了一个月的pro，不知道有没有被宰。

工作与学习复盘报告

日期： 2026年5月6日
状态： 五一假期结束，回归正常工作节奏

一、 实习工作：智慧课堂系统测试

1. “手动评价”任务测试反馈

  • 问题现象：
    • 教师在执行“手动评价”时无法改分。
    • 评价栏内未正常呈现 AI 评价语句。
  • 逻辑复盘：
    • 起初误判为“课堂 AI 评价”功能故障，后经确认该功能属于**“去评价”**模块（此前本人未接触过的新功能）。
    • 虽然复现难度低，但该功能与 AI 评价逻辑深度耦合，存在较多无固定逻辑的随机 Bug。

2. 典型随机 Bug（病毒 Bug）汇总

类别 现象描述 观察结论
课件播放 课程中心 PPT 无法播放。 具有随机性，重新上传可修复，有时会自动恢复。
入口重定向 输入 zcat.cn 随机跳转至学生登录页，而 c.zcat.cn 正常。 困扰已久，在无痕模式或刷新浏览器时随机出现。

3. 系统思辨：复杂性 vs 普及性

  • 复杂性痛点： 预览报告生成、分数显示等操作链路过长；预览报告更新时间与模板生成时间过长且不可控。
  • 普及性缺口： 现有教程文档（如超喵 AI 群）未覆盖全功能，导致基层教师（如附小老师）因操作繁琐或认知门槛而弃用。

二、 毕业设计进展

1. 课题核心逻辑

题目： 《基于强化学习的城镇开发边界模拟优化与空间规划方案自动生成》

  • 环境 (Environment)：
    • 基础数据： 人口密度、地形特点、周边商业、战略目标。
    • 硬约束规则： 耕地红线、坡度 $> 25^{\circ}$ 禁建区、生态敏感区。
  • 机制 (Mechanism)：
    • 通过 动作-奖励-策略 机制进行训练。
    • 目标： 在“耕地保护”与“经济发展”之间寻找帕累托最优，实现土地资源分配自动化。

2. 技术实现与进展

  • 数据处理： 在 Cursor 的辅助下，今日已完成毕设全部数据的清洗与处理工作。
  • API 实验： 尝试将 DeepSeek API 接入之前的“信访项目”，目前处于初步调通阶段。
  • 成本与投入：
    • 投入 10 元用于 DeepSeek API 余额。
    • 订阅了 Cursor Pro（约 40r/月），旨在加速代码产出。
  • 明日计划： 正式开启智能体（Agent）的训练测试。
